Shell Quest Engine

Terminal 2D game engine — developer documentation.

Shell Quest is a data-driven engine for terminal games. Mods are plain YAML packages; the engine loads them, validates their manifest, and runs a fixed-timestep game loop driving three systems: Animator, Compositor, and Renderer.

Contents

  • App — entry point binary; creates ShellEngine, points to mod source, calls run()
  • Engine — core runtime: game loop, ECS world, systems pipeline
    • Rendering Pipeline — animator → compositor → renderer
    • World API — TypeMap component registry
    • Events — EngineEvent variants
  • Schemas — YAML contracts for mod and scene files
    • Scene Schema — every field documented
    • Effects Catalogue — built-in visual effects
  • Mod: Shell Quest — reference mod demonstrating the stage/effect system

Project layout

app/                  <- entry point binary
engine/               <- core library crate
  src/
    lib.rs            <- ShellEngine, game_loop
    scene.rs          <- Scene, Layer, Stage, Effect types
    effects.rs        <- built-in effect implementations
    buffer.rs         <- terminal cell buffer
    world.rs          <- TypeMap ECS world
    events.rs         <- EngineEvent queue
    systems/
      animator.rs     <- stage lifecycle advancement
      compositor.rs   <- scene + layer rendering to buffer
      renderer.rs     <- buffer -> terminal via ratatui
mod/
  shell-quest/        <- reference mod
    mod.yaml
    scenes/
      intro.yml
schemas/
  mod.schema.yaml
  scene.schema.yaml
