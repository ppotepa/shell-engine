Overview

The EventQueue collects EngineEvent values during input polling and drains them at the start of each game loop tick. Systems may push additional events (e.g. SceneTransition) during their execution.

Drain-per-tick pattern: at the top of each game loop iteration, all queued events are taken out in one call to drain(). The queue is empty for the remainder of that tick. New events pushed by systems during the tick are drained on the next iteration.

// Drain all events at start of tick
let events = world.get_mut::<EventQueue>().unwrap().drain();

for event in events {
    match event {
        EngineEvent::Quit => { /* exit game loop */ }
        EngineEvent::KeyPressed(code) => { /* handle key */ }
        EngineEvent::SceneTransition { to_scene_id } => { /* load next scene */ }
        // ...
    }
}

EngineEvent variants

VariantFieldsDescriptionSource
Tick — Fired once per game loop iteration (every ~16 ms). Used to drive time-based logic. game_loop
KeyPressed KeyCode A key was pressed. Excludes Esc and q, which produce Quit instead. game_loop (crossterm)
Quit — User pressed Esc or q. The game loop exits on this event. game_loop (crossterm)
SceneTransition to_scene_id: String Requests a transition to the named scene. The game loop will call SceneLoader.load_by_id, call world.clear_scoped(), and register the new scene and a fresh Animator. animator_system, game_loop
SceneLoaded scene_id: String Reserved for future use. Emitted after a scene is successfully loaded and registered. —
TerminalResized width: u16, height: u16 The terminal was resized. The game loop recreates the Buffer with the new dimensions. game_loop (crossterm)

EventQueue API

MethodSignatureDescription
new() () -> EventQueue Creates an empty queue.
push() (&mut self, EngineEvent) Appends an event to the queue.
drain() (&mut self) -> Vec<EngineEvent> Takes all queued events, leaving the queue empty. Returns owned Vec.
is_empty() (&self) -> bool Returns true when no events are pending.
