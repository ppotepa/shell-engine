Overview

The engine crate is the core runtime library. It loads a mod, validates its manifest, enters an ECS-style game loop, and drives three systems every tick: Animator, Compositor, and Renderer.

Mods are data-only packages (directory or .zip) containing a mod.yaml manifest and one or more *.yml scene files.

System pipeline

  +-------------------------------------------------------------+
  |                        game_loop()                          |
  |                                                             |
  |  INPUT          -> EventQueue                               |
  |                         |                                   |
  |                         v                                   |
  |  animator_system()  advances SceneStage, step_idx           |
  |                         |                                   |
  |                         v                                   |
  |  compositor_system() reads Scene + Animator, writes Buffer  |
  |                         |                                   |
  |                         v                                   |
  |  renderer_system()  flushes Buffer -> terminal (ratatui)    |
  +-------------------------------------------------------------+

Components registered in World

TypeScopeDescription
EventQueueSingletonCollects and drains engine events each tick.
BufferSingletonTerminal cell buffer. Recreated on resize.
TerminalRendererSingletonRatatui terminal handle. Owns raw-mode lifecycle.
SceneLoaderSingletonLoads scene YAML files by id or path.
SceneScopedCurrently active scene (cleared on transition).
AnimatorScopedStage/step/elapsed state for the active scene.

Scoped components are cleared via world.clear_scoped() on every scene transition.

Scene lifecycle

Each scene passes through three stages managed by Animator:

StageTransition intoTransition out
OnEnterScene loadAll steps complete -> OnIdle
OnIdleAutomaticTrigger fires (e.g. any-key) -> OnLeave
OnLeaveTriggerAll steps complete -> Done + SceneTransition event
DoneInternalFires SceneTransition with scene.next
