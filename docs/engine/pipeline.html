Overview

Every 16 ms the engine runs three systems in sequence. Each system reads from the World and may mutate exactly one resource to avoid Rust borrow conflicts.

INPUT EVENTS
     |
     v
+----------------------------------------------------------+
|  1. animator_system()                                    |
|     Reads:  Scene (stages), Animator                     |
|     Writes: Animator (stage, step_idx, elapsed_ms)       |
|             EventQueue (SceneTransition when Done)       |
+----------------------------------------------------------+
     |
     v
+----------------------------------------------------------+
|  2. compositor_system()                                  |
|     Reads:  Scene (layers, stages), Animator             |
|     Writes: Buffer (cells)                               |
|                                                          |
|     a) Snapshot all scene data (immutable borrows done)  |
|     b) buffer.fill(bg_colour)                            |
|     c) render_layer_base -- write text glyphs to buffer  |
|     d) apply_layer_effects -- per-layer stage effects    |
|     e) apply scene-level stage effects (full buffer)     |
+----------------------------------------------------------+
     |
     v
+----------------------------------------------------------+
|  3. renderer_system()                                    |
|     Reads:  Buffer.dirty, Buffer.cells                   |
|     Writes: Terminal (ratatui draw call)                 |
|             Buffer.dirty = false                         |
+----------------------------------------------------------+

1 — animator_system

Advances Animator.elapsed_ms by 16 ms each tick. When the current step's duration is exceeded (and looping is false), the animator either advances to the next step or transitions to the next stage.

Stage transitionConditionSide effect
OnEnter -> OnIdleAll on_enter steps complete—
OnIdle -> OnLeaveTrigger fires (e.g. any-key key event)—
OnLeave -> DoneAll on_leave steps completePushes SceneTransition

Looping steps (loop: true) never advance the step index automatically. They continue until the stage trigger fires or a scene transition is forced.

2 — compositor_system

Composites the active scene into the Buffer. All immutable reads from Scene and Animator are snapshotted before taking the mutable buffer reference, avoiding Rust borrow conflicts.

Compositing order

StepWhat happens
1. Fill backgroundbuffer.fill(scene.bg_colour) — blanks every cell.
2. Layer base contentWrites layer glyphs (text) at (x, y) with fg/bg colour.
3. Layer effectsApplies per-layer stage effects to the layer's row region.
4. Scene effectsApplies scene-level stage effects to the full buffer region.

Effect progress

Progress is computed as elapsed_ms / step_duration_ms, clamped to 0.0..=1.0, then passed through the step's easing function before being handed to each effect implementation.

3 — renderer_system

Skips the draw call when Buffer.dirty is false. Otherwise, iterates all cells and writes them into ratatui's frame buffer, then calls terminal.draw() to flush to the terminal.

After drawing, buffer.mark_clean() is called to reset the dirty flag.

Borrow checker contract

Each system follows a strict pattern to satisfy the Rust borrow checker:

// Pattern used in compositor_system and animator_system
let snapshot = {
    let scene    = world.get::<Scene>().unwrap();    // immutable borrow
    let animator = world.get::<Animator>().unwrap(); // immutable borrow
    // ... clone / copy all needed data ...
}; // borrows released here

// Now safe to take mutable borrow
let buffer = world.get_mut::<Buffer>().unwrap();
