# docs

Supporting documentation assets and archived reference material.

## Purpose

`docs/` is now mostly a holding area for generated site assets, imported older
documents, and media used by documentation. The current canonical technical
guides are the root docs and the local `README.md` / `README.AGENTS.MD` files
next to each subsystem.

## Subdirectories

- `ARCHIVED/` — older imported or superseded notes kept for reference
- `assets/` — legacy markdown documents and supporting doc material
- `html/` — generated static documentation site assets

Use this directory for reference material, not as the primary place to learn
the current architecture.
