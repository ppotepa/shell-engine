Core Concepts

  • Scene: A static scene definition in the core model. Compiled from YAML, contains layers, sprites, behaviors, effects.
  • SceneRuntime: The live runtime representation of a Scene in the World. Tracks sprite visibility, animation state, behavior variables.
  • Sprite: A visual element (image or text). Properties: position, size, animation, effects.
  • Layer: Z-ordered container for sprites. Scenes have multiple layers for depth organization.
  • BehaviorSpec: A behavior script definition, compiled from YAML. Defines conditions and commands.
  • BehaviorCommand: Execution primitive: Move, SetVisibility, Animate, Transition, etc.
  • Effect: Visual transformation: fade, crt-reflection, glow, contrast, color-remap.
  • Animation: Timed state change of a sprite or layer (appearance, position, effects).
  • World: The runtime container for active scene, sprites, layers, systems, and asset cache.
  • EngineEvent: Top-level event enum: KeyPressed, MouseMoved, Tick, Quit, SceneTransition, ObjectCommand.

Systems & Pipeline

  • Game Loop: Fixed-timestep main loop (~60 FPS). Polls events, updates systems, handles display.
  • Animator: System that tracks animation timing, manages sprite visibility transitions.
  • Compositor: Calculates final layout: layer z-order, sprite positioning, culling.
  • Renderer: Converts composite into terminal raster buffer (character grid + colors).
  • BehaviorSystem: Executes behavior scripts: condition checks, command execution.
  • SceneLifecycleManager: Manages scene transitions, stage progression, terminal-shell input.
  • MenuSystem: Handles menu navigation and selection.
  • AudioSystem: Manages audio playback.
  • AssetLoader: Loads and caches assets: images, fonts, objects.

Authoring & Compilation

  • YAML: Scene authoring format (scenes/*.yml). Defines layers, sprites, behaviors, effects, transitions.
  • Compilation: Transform YAML scene into runtime Scene. Handles object references, template expansion, normalization.
  • Cutscene: Timed image sequence. Compiled into sprite visibility keyframes.
  • Object Reference: Scene-level reusable object template (scenes/objects/*.yml). Can be included with `ref:` + `with:` overrides.
  • Input Profile: Scene-level input handling mode: 'menu', 'terminal-shell', 'obj-viewer', etc.
  • Terminal Shell: Interactive line-editing interface within a scene. Provides `help`, `clear`, `ls`, `pwd`, `echo`, `whoami` commands.

Tools & Formats

  • schema-gen: Tool that generates YAML schemas for all mods. Validates schema drift.
  • devtool: CLI for scene/asset creation: `create mod`, `create scene`, `create sprite`, `edit sprite`.
  • Manifest: YAML file describing a mod: scenes, assets, global settings.
  • PNG: Image format (indexed-color). Supported bit depths: 1, 2, 4-bit.
  • TTF: Font format. Rasterized on-demand to bitmap glyphs.
  • OBJ: 3D geometry format. Converted to ASCII art for terminal rendering.
  • Dithering: Floyd-Steinberg optional algorithm for smooth color reduction.

Development Concepts

  • Hot Reload: Editor automatically refreshes when files change on disk.
  • Virtual Buffer: Off-screen rendering target. Scaled to terminal size on output.
  • Halfblock Mode: Character rendering using ▄ and ▀ for 2x vertical resolution.
  • Braille Mode: Character rendering using braille patterns ⠿ for fine-grained pixel control.
  • Cell Mode: Character-level rendering (one color per cell).
  • RAII: Resource Acquisition Is Initialization. Used for mouse capture guard cleanup.
  • Inference: Information deduced from code analysis (not explicitly confirmed).
