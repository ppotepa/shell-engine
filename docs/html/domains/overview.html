Key domains

  • Launcher / app — entrypoint binaries (app/)
  • Runtime engine — game loop, systems, renderer (engine/)
  • Core model — authored scene model and effects (engine-core/)
  • Authoring — compile/normalize pipeline (engine-authoring/)
  • Editor — TUI authoring tool and preview (editor/)
  • Mods — content and assets (mods/)
  • Tools — schema-gen, devtool, converters (tools/)

Responsibilities

Each domain has clear responsibilities: engine runs scenes, engine-core defines the model, authoring transforms YAML to runtime, editor provides UI for authoring, mods contain content.
