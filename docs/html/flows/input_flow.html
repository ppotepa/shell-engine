Key input processing (confirmed)

  1. crossterm event poll in game_loop
  2. Key/Mouse/Resize → EngineEvent (KeyPressed(KeyEvent) preserves modifiers)
  3. Push events to world queue
  4. SceneLifecycleManager.process_events routes to:
    • Terminal-shell handler (if active input profile)
    • Menu handler
    • Obj-viewer handler
  5. Terminal-shell maps KeyEvent → tui_input::InputRequest
  6. tui_input edits prompt buffer and emits commands on Return
