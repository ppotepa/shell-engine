Confirmed pipeline (from engine/src/game_loop.rs)

  1. Input polling (crossterm): Key/Mouse/Resize -> EngineEvent pushed
  2. Emit EngineEvent::Tick
  3. Drain events and pass to SceneLifecycleManager::process_events
  4. Run systems:
    • animator_system (may emit transitions)
    • drain post-animator events and process them (confirmed)
    • behavior_system
    • audio_system
    • compositor_system
    • renderer_system
  5. Sleep to maintain target FPS

Notes

Post-animator drain is an explicit design to avoid rendering an intermediate "done" frame that could briefly re-show sprites. This is confirmed by code comments in game_loop.rs.
