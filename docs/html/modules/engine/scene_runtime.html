Overview

engine/src/scene_runtime.rs contains SceneRuntime, the materialized runtime view of a Scene with stable object ids, behavior bindings, object_states, and optional TerminalShellState.

Key responsibilities

  • Materialize authored Scene into runtime objects and ids
  • Track per-object runtime state and behaviors
  • Provide per-frame mutable runtime state (e.g., TerminalShellState)

TerminalShellState (confirmed)

When a scene defines an input profile terminal-shell, SceneRuntime holds a TerminalShellState that contains:

  • controls — TerminalShellControls from the authored scene
  • input — tui_input::Input buffer
  • output_lines, history, history cursor
Excerpt: struct TerminalShellState { controls: TerminalShellControls, input: Input, output_lines: Vec, history: Vec, history_cursor: Option, }

Execute command (confirmed)

The runtime includes built-in commands (confirmed in code): clear, help, pwd, whoami, echo, and supports custom commands declared in scene YAML.
