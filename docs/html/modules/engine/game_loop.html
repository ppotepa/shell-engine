Summary

Confirmed in engine/src/game_loop.rs. The engine uses a fixed-timestep loop. Input is polled via crossterm, converted to EngineEvent variants, and pushed into the world's event queue. A Tick event is emitted each frame, events are drained and processed by SceneLifecycleManager, and systems run in a fixed order.

Responsibilities

  • Poll input and terminal events (keys, mouse, resize)
  • Push EngineEvent::Tick and other EngineEvents
  • Drain events and call SceneLifecycleManager::process_events
  • Run systems in fixed order
  • Process post-animator events in same frame (confirmed)
