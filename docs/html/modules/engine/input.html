Design

Input uses crossterm polling at the game_loop level to receive raw KeyEvent/Mouse events. The engine changed from KeyCode-only to full KeyEvent to preserve modifiers and key-kind through lifecycle processing (confirmed).

Terminal-shell mapping

Terminal-shell uses tui-input (core only) for editing the prompt. The engine maps KeyEvent to tui_input::InputRequest implementing common editing shortcuts (Ctrl-A/E/K/U, word motion, arrows, delete/backspace).
