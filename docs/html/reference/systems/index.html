Engine Systems (9 pages)

Runtime systems that orchestrate the frame pipeline. Listed in typical execution order within a frame.

  • Animator — Animation timing and sprite visibility
  • Compositor — Layout and z-order calculation
  • Renderer — Terminal buffer rendering
  • BehaviorSystem — Behavior execution
  • AudioSystem — Audio playback
  • SceneLifecycleManager — Scene transitions and lifecycle
  • MenuSystem — Menu navigation and selection
  • GameLoop — Fixed-timestep main loop
  • AssetLoader — Asset loading and caching

Frame Pipeline Order

  1. Poll events (input, timing)
  2. Scene Lifecycle (transitions, terminal-shell)
  3. Menu System (if active)
  4. Behavior System (execute commands)
  5. Animator (update timing, emit transitions)
  6. Compositor (calculate layout)
  7. Renderer (output frame)
