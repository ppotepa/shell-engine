Per-Frame Execution Order

  1. Event Poll: Check terminal/mouse input → EngineEvent queue
  2. Scene Lifecycle: Process scene transitions, terminal-shell input
  3. Behavior System: Execute BehaviorCommand for each sprite/layer
  4. Animator: Update animation timing, sprite visibility
  5. Compositor: Calculate layout, z-order, layer visibility
  6. Renderer: Convert composite to terminal buffer
  7. Display: Output to terminal

Key Invariants

  • Behavior state reset before behavior execution
  • Animator emits scene transitions processed same frame
  • Compositor runs after all state mutations
  • Renderer is read-only (no side effects)

Timestep

Fixed 60 FPS timestep in game_loop.rs. Frame timing managed by TerminalEvent::Tick.
