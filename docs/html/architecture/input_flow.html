Input Sources

  • Terminal: Raw keyboard input (KeyCode → KeyEvent with modifiers)
  • Mouse: Terminal mouse events (if supported)

Processing Pipeline

  1. Terminal Poll: Read raw bytes, convert to KeyEvent (crossterm)
  2. Engine Event Queue: KeyPressed(KeyEvent) → EngineEvent
  3. Scene Lifecycle: Check input profile (terminal-shell, menu, obj-viewer, etc)
  4. Terminal Shell: If active, consume keystrokes for command editing
  5. Menu System: Process menu navigation/selection
  6. Behavior System: Dispatch to scene behaviors (if any)

Terminal Shell Input Profile

Provides line editing: Ctrl+A/E/K/U/W, arrow keys, delete/backspace, word motion. Consumes keys before menu transitions.

Key Event Modifiers

KeyEvent preserves Shift/Ctrl/Alt/Meta. Allows modifier-aware behavior scripting.
