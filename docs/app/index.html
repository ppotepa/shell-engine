Overview

The app crate is the entry point binary. It creates a ShellEngine instance, points it at the mod source directory, and calls run() to hand control to the engine.

Flow

  1. Application starts.
  2. ShellEngine is constructed with the relative path mods/shell-quest/ (or SHELL_QUEST_MOD_SOURCE override).
  3. The engine validates mod.yaml against the mod schema.
  4. engine.run() enters the game loop and does not return until the user quits.

Launcher

Use ./launcher.sh to start the game in a new terminal window (GNOME/KDE/Xfce/Kitty/Alacritty/xterm when available). It reads terminal hints from mod.yaml and exports SHELL_QUEST_MOD_SOURCE before launch.

For game-like wrapper mode use ./launcher.sh --game-window (fullscreen/minimal chrome where supported), force a specific emulator with --terminal konsole, or use --normal-window for a regular terminal frame.

Resolution strategy

Runtime supports optional fixed virtual buffer configured in mod.yaml (terminal.use_virtual_buffer, terminal.virtual_size, terminal.virtual_policy).

When enabled, scene rendering happens in virtual space first, then output is translated to the real terminal size. Policy fit uses adaptive scale + letterbox; strict requires terminal at least as large as virtual size.
