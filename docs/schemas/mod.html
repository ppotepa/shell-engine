Overview

mod.yaml is the root manifest loaded by the engine. It defines mod identity, entrypoint scene, and terminal/runtime requirements.

Current supported fields

FieldTypeRequiredDescription
namestringyesHuman-readable mod name.
versionstringyesMod version.
entrypointstringyesScene file path (e.g. /scenes/intro.logo.yml).
terminal.min_coloursintegernoMinimum color capability required.
terminal.min_widthintegernoMinimum terminal width in columns.
terminal.min_heightintegernoMinimum terminal height in rows.
terminal.target_fpsintegernoTarget FPS lock (default 60).
terminal.target-fpsintegernoAlias of target_fps.
terminal.use_virtual_bufferbooleannoEnable fixed virtual framebuffer before present.
terminal.virtual_sizestringnoVirtual size in WIDTHxHEIGHT (e.g. 320x240).
terminal.virtual_policystringnostrict (no downscale) or fit (adaptive fit + letterbox).

Current example

name: Shell Quest
version: 0.1.0
entrypoint: /scenes/intro.logo.yml
terminal:
  min_colours: 256
  min_width: 220
  min_height: 55
  use_virtual_buffer: true
  virtual_size: "220x55"
  virtual_policy: strict

Virtual buffer behavior

  • use_virtual_buffer: false — engine renders directly to real terminal buffer.
  • use_virtual_buffer: true — engine renders to virtual buffer, then translates to terminal output.
  • virtual_policy: strict — requires terminal at least as large as virtual size (startup check enforces).
  • virtual_policy: fit — fits virtual output into terminal with centered letterbox when needed.

Notes about schema aliases

  • target-fps is an alias of target_fps.
  • use-virtual-buffer is an alias of use_virtual_buffer.
  • virtual-size is an alias of virtual_size.
  • virtual-policy is an alias of virtual_policy.
