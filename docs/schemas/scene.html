Overview

Each scene is a .yml file in a mod's scenes/ directory. Scenes define a visual presentation composed of layers and a stage lifecycle that drives effects via on_enter -> on_idle -> on_leave.

The schema is enforced by schemas/scene.schema.yaml (JSON Schema 2020-12). All fields not marked required have safe defaults and may be omitted.

Minimal example

id: my-scene
title: My Scene

Full example

id: intro
title: Intro
cutscene: true
bg_colour: black
stages:
  on_enter:
    steps:
      - effects:
          - name: crt-on
            duration: 5000
            params:
              easing: ease-out
  on_idle:
    trigger: any-key
    steps:
      - effects:
          - name: scanlines
            duration: 2000
            loop: true
  on_leave:
    steps:
      - effects:
          - name: power-off
            duration: 1500
layers:
  - type: text
    content: "Hello"
    x: 4
    y: 4
    fg_colour: white
    stages:
      on_enter:
        steps:
          - effects:
              - name: fade-in
                duration: 1000
next: null

Scene root fields

FieldTypeRequiredDefaultDescription
idstringrequired—Unique scene identifier used for transitions.
titlestringrequired—Human-readable scene title.
cutscenebooleanoptionalfalseMarks the scene as non-interactive.
bg_colourcolouroptionalblackBackground colour filling the entire buffer on load.
stagesSceneStagesoptionalempty stagesLifecycle stages for the scene. See SceneStages below.
layersLayer[]optional[]Ordered list of content layers. See Layer below.
nextstring | nulloptionalnullScene id to load after on_leave completes.

SceneStages

Contains three lifecycle stage objects keyed by phase name.

KeyDescription
on_enterPlays once when the scene loads. Advances to on_idle when all steps complete.
on_idleRuns while the scene is active. A trigger causes transition to on_leave.
on_leavePlays once when leaving. Fires SceneTransition when all steps complete.

Stage

FieldTypeRequiredDefaultDescription
triggerstringoptionalnoneOnly on on_idle. Allowed: any-key, timeout, none.
stepsStep[]optional[]Sequential list of steps. Each step completes before the next begins.

Step

All effects within a step execute in parallel. Steps within a stage execute sequentially.

FieldTypeRequiredDescription
effectsEffect[]requiredOne or more effects that run simultaneously. The step ends when the longest-duration effect finishes.

Effect

FieldTypeRequiredDefaultDescription
namestringrequired—Effect identifier. See Effects Catalogue.
durationinteger (ms)required—Duration in milliseconds. Must be 0 when loop: true.
loopbooleanoptionalfalseWhen true the effect repeats indefinitely and never advances the step.
paramsobjectoptional—Optional parameters. See params.easing and params.colour.
params.easingstringoptionallinearProgress easing. One of: linear, ease-in, ease-out, ease-in-out.
params.colourcolouroptional—Target colour for colour-based effects (e.g. clear-to-colour).

Layer

Currently only type: text is supported.

FieldTypeRequiredDefaultDescription
typestringrequired—Layer type. Must be text.
contentstringrequired—Text string to render.
xintegeroptional0Column offset from left edge.
yintegeroptional0Row offset from top edge.
z_indexintegeroptional0Render order. Higher values render on top (reserved for future use).
fg_colourcolouroptionalwhiteForeground glyph colour.
bg_colourcolouroptionalscene bg_colourBackground cell colour. Falls back to scene background.
stagesLayerStagesoptionalemptyPer-layer lifecycle stages with independent effect steps.

Colour values

Valid colour names for bg_colour, fg_colour, and params.colour:

black  white  gray  red  green  blue  yellow  cyan  magenta
