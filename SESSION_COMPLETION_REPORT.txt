DEPENDENCY INVERSION BREAKTHROUGH
─────────────────────────────────
Problem: Renderers wanted to move to engine-compositor, but they:
  1. Depended on ModAssetSourceLoader (engine-specific)
  2. ModAssetSourceLoader depended on repositories (engine-specific)
  3. Moving both would create circular dependency

Solution: Extract ALL assets to engine-asset before moving renderers
  ✓ engine-asset has NO engine dependencies (only engine-core, authoring)
  ✓ Renderers can now safely depend on engine-asset
  ✓ No circular dependencies possible

LESSON: When crate extraction is blocked, look for dependency inversion
        Extract shared dependencies FIRST, then move systems.

ARCHITECTURAL PATTERN: "Levels of Abstraction"
───────────────────────────────────────────────
1. engine-core: Concrete types (Scene, Buffer, effects)
2. engine-asset: Infrastructure (loading, repositories)
3. Domain crates: Systems (behavior, scene-runtime, compositor)
4. engine: Orchestrator (ties everything together)
5. app: Entry point

Each level depends only on lower levels, never upward.
