Terminal and viewport sizing

This page describes how terminal size requirements and virtual resolution work in Shell Quest.

Key fields

  • terminal.min_width and terminal.min_height define the minimum terminal size required to start the mod.
  • terminal.use_virtual_buffer enables rendering into a virtual buffer before presenting to the real terminal.
  • terminal.virtual_size defines the virtual buffer size. It accepts either WIDTHxHEIGHT or max-available.
  • terminal.virtual_policy controls how a virtual buffer is presented. Current values are fit and strict.
  • virtual-size-override in a scene overrides the mod-level virtual size for that one scene.

How it is resolved

  1. The engine checks min_width and min_height before startup.
  2. The mod-level virtual_size is used unless a scene declares virtual-size-override.
  3. If the resolved size is max-available, the virtual buffer matches the real terminal size and updates on resize.
  4. If the resolved size is fixed and the policy is fit, the engine preserves aspect ratio and centers the viewport with letterboxing.
  5. If the resolved size is fixed and the policy is strict, the engine uses the exact virtual canvas and centers any cropped or unused area.

Recommended default for Shell Quest

This is the current recommended setup when we want a minimum authored terminal size, but still want the mod to fill larger terminals:

terminal:
  min_colours: 256
  min_width: 237
  min_height: 65
  use_virtual_buffer: true
  virtual_size: max-available
  virtual_policy: fit

Important: min_width and min_height only block startup below the required size. They do not stretch the scene. Filling the screen comes from resolving the virtual size to the real terminal, which is what max-available does.

Examples

1. Minimum size plus always fill the current terminal

terminal:
  min_width: 237
  min_height: 65
  use_virtual_buffer: true
  virtual_size: max-available
  virtual_policy: fit

Use this for the main Shell Quest flow. The mod still requires at least 237x65, but once it starts, it fills the current terminal and keeps following terminal resizes.

2. Fixed cinematic canvas with centered letterbox

terminal:
  use_virtual_buffer: true
  virtual_size: "320x240"
  virtual_policy: fit

Use this when a scene is authored for a specific aspect ratio and should stay centered instead of stretching.

3. Fixed exact canvas for strict authored layouts

terminal:
  use_virtual_buffer: true
  virtual_size: "237x65"
  virtual_policy: strict

Use this when the authored layout is intended to stay exact. The terminal must be large enough for the fixed virtual size or startup checks should fail.

4. Scene-specific override for one cinematic scene

id: intro.logo
title: Intro Logo
virtual-size-override: "320x240"

This overrides the mod-level virtual size only for one scene. It is useful for a stylized intro, but it will usually reintroduce a centered viewport instead of full fill.

Current project rule

Shell Quest now defaults to mod-level virtual_size: max-available and the main intro/menu scenes do not declare virtual-size-override. That keeps the authored minimum terminal requirement while letting the scene fill large terminals.
